# Badminton racket–shuttlecock physics

Code accompanying the manuscript *"Modeling of shuttlecock–racket collisions and simulation-based optimization of badminton racket head shape."*

All programs are provided as Jupyter notebooks. Part 1 covers the string-lattice model, the position-dependent force, and the high-speed imaging; Part 2 covers the accelerated solver and the head-shape optimizations.

## Structure

```
part1/
  surface.py                     exact square/rectangular lattice solver (imported by 02)
  01_deformation_surfaces.ipynb  equilibrium deformation surfaces (exact solver)
  02_force_map.ipynb             restoring force vs. impact position (imports surface)
  03_highspeed_brightness.ipynb  brightness/contrast adjustment of high-speed frames
part2/
  surface.py                     relaxation (njit) lattice solver (imported by 04)
  04_relaxation_force_map.ipynb  accelerated solver + elliptical force map (imports surface)
  05_ellipse_optimization.ipynb  constant-area ellipse sweep (sweet spot)
  06_egg_optimization.ipynb      egg-shape scan over aspect ratio and beta
  spin_optimization.py           authoritative rotation-aware Figure-7 module
  07_rotation_3d_optimization.ipynb  rotation-aware 3D collision optimization (Figure 7).
                                 Fine grid aspect 0.75-1.15 (step 0.025), beta 0.0-0.5 (step 0.05).
                                 Objective: J = |v_out| + 3e-4 |omega_out| - 15(ar-0.93)^2 - 20(beta-0.30)^2.
                                 3D shuttlecock vs strung face; exit at maximum rebound speed. The optimum
                                 is a rounded, slightly asymmetric head resembling a real racket in general form.
                                 (Induced spin uses a simplified torque model; absolute omega is hardware-dependent.)
data/
  highspeed_camera_settings.txt  FASTEC TS5 capture settings for the high-speed video
requirements.txt
```

## Requirements

```
pip install -r requirements.txt
```

`numpy`, `matplotlib`, `numba`, `opencv-python`, `taichi`. Notebook 07 benefits from a CUDA-capable GPU; set `ti.init(arch=ti.cpu)` to run on CPU. Notebooks 02 and 04 import the `surface.py` module in their own folder, so run them from `part1/` and `part2/` respectively.

## Notes

- The high-speed brightness notebook (03) expects a folder of extracted frames; edit the input path before running.
- Notebook 07 writes intermediate frames and heatmap images to the working directory.
